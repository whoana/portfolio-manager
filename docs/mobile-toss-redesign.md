## 1. 리서치 요약: 토스앱 디자인 핵심 원칙

### 1.1 디자인 철학

| 원칙 | 설명 | 출처 |
|------|------|------|
| **한 화면, 한 기능** | 사용자가 화면에 헤매는 시간을 최소화. 한 화면에서 하나의 작업에만 집중 | [토스 증권 UX](https://brunch.co.kr/@kellypoly/36) |
| **기능은 단순하게, 용어는 쉽게** | 금융 용어를 일상 언어로 치환. 초보자 중심 설계 | [토스증권 UX/UI 분석](https://brunch.co.kr/@plusx/71) |
| **큰 숫자, 작은 라벨** | 핵심 금액은 크고 굵게, 부가 정보는 작은 서브텍스트로 | [토스 UX 법칙 분석](https://brunch.co.kr/@uniher/18) |
| **친근한 미니멀리즘** | 불필요한 장식 없이 정보에 집중하되 친근한 톤 유지 | [토스 디자인 시스템](https://toss.tech/article/toss-design-system) |

### 1.2 TDS(Toss Design System) 핵심 컴포넌트

토스 공식 문서([TDS Components](https://developers-apps-in-toss.toss.im/design/components.html))에서 제공하는 11개 핵심 컴포넌트:

| 컴포넌트 | 용도 | 우리 앱 적용 |
|----------|------|-------------|
| **ListRow** | 목록 항목 (아이콘 + 텍스트 + 우측 값) | 종목 리스트, 성장 전망 행 |
| **Navigation** | 하단 탭 바 | 모바일 하단 내비게이션 |
| **BottomCTA** | 하단 고정 액션 버튼 | 종목 추가, 저장 등 |
| **Tab** | 탭 네비게이션 | 포트폴리오 전환 |
| **Badge** | 상태 라벨 | 카테고리 뱃지 (배당/성장/안전판) |
| **Button** | CTA 버튼 | 전체 시세 갱신, Excel 내보내기 |
| **Top** | 상단 히어로 영역 | 자산 요약 카드 |
| **Paragraph** | 본문 텍스트 | 안내 문구, 설명 |
| **Border** | 구분선 | 섹션 구분 |

### 1.3 토스 컬러 시스템 특징

토스는 OKLCH 기반 컬러 시스템을 사용하며, [7년만의 컬러 시스템 업데이트](https://toss.tech/article/tds-color-system-update)에서 다음 원칙을 따름:

- **배경**: 순백(#FFFFFF) 또는 연한 회색(#F4F4F4)
- **주요 액센트**: 토스 블루(#3182F6)
- **텍스트**: 검정(#191F28) / 회색(#8B95A1) / 연회색(#B0B8C1)
- **성공/수익**: 빨강(#F04452) — 한국 주식 상승색
- **하락/경고**: 파랑(#3182F6) — 한국 주식 하락색
- **구분선**: 연한 회색(#EDEFF1)
- **카드 그림자**: `0 2px 8px rgba(0,0,0,0.04)` 수준의 미세한 그림자

## 2. 현재 앱 vs 토스 스타일 비교

### 2.1 현재 앱 구조 (12개 컴포넌트)

```
page.tsx (메인 페이지)
├── IntroPage.tsx (인트로)
├── PortfolioTable.tsx (종목 구성 - 데스크톱 테이블 + 모바일 카드)
├── AllocationTable.tsx (투자 배분 - 데스크톱 테이블 + 모바일 카드)
├── PortfolioSummary.tsx (요약 대시보드)
├── GrowthReport.tsx (자산성장 전망)
├── AddStockModal.tsx (종목 추가/수정 모달)
├── StockSearch.tsx (종목 검색 자동완성)
├── SettingsBar.tsx (테마/도움말 토글)
├── ThemeProvider.tsx (테마 컨텍스트)
├── HelpProvider.tsx (도움말 컨텍스트)
├── ClientProviders.tsx (프로바이더 래퍼)
└── BalloonTooltip.tsx (도움말 풍선)
```

### 2.2 핵심 차이점

| 항목 | 현재 | 토스 스타일 목표 |
|------|------|-----------------|
| **헤더** | 컬러 배경(primary) sticky 헤더 | 흰 배경, 미니멀 타이틀, 우측 아이콘 |
| **메인 화면 상단** | 포트폴리오 탭 즉시 노출 | **자산 총액 히어로 카드** (큰 숫자) |
| **숫자 크기** | text-sm ~ text-base | 핵심 금액 **text-2xl ~ text-3xl** |
| **카드 스타일** | border + rounded-xl | **그림자 + rounded-2xl**, border 최소화 |
| **종목 리스트** | 모바일: 카드 내 메타 정보 | **ListRow 패턴**: 로고+이름 | 수익률+금액 |
| **하단 내비** | 4탭(종목/배분/요약/성장) | 유지하되 **아이콘 더 크게, 라벨 간결** |
| **색상** | 앰버(Claude)/네이비(Classic)/다크 | **토스 블루 + 순백** 신규 테마 추가 |
| **여백** | px-3~5, py-2~4 (촘촘) | **px-5~6, py-4~6** (여유로운 여백) |
| **모달** | Bottom sheet (부분 적용) | **Full bottom sheet** + 드래그 핸들 |
| **구분선** | border 기반 | **배경색 차이** (섹션 간 회색 갭) |

## 3. 변경 범위 상세

### 3.1 신규 "Toss" 테마 추가 (globals.css)

**작업량: 소 (1시간)**

```css
/* 토스 스타일 라이트 테마 */
[data-theme="toss"] {
  --color-primary: 49 130 246;       /* #3182F6 토스 블루 */
  --color-primary-hover: 29 110 226;
  --color-primary-fg: 255 255 255;
  --color-background: 244 244 244;   /* #F4F4F4 연회색 배경 */
  --color-foreground: 25 31 40;      /* #191F28 */
  --color-card-bg: 255 255 255;
  --color-card-border: 237 239 241;  /* #EDEFF1 매우 연한 */
  --color-muted: 176 184 193;        /* #B0B8C1 */
  --color-muted-fg: 139 149 161;     /* #8B95A1 */
  --color-accent-green: 240 68 82;   /* #F04452 한국식 상승=빨강 */
  --color-accent-green-bg: 255 243 244;
  --color-accent-red: 49 130 246;    /* #3182F6 한국식 하락=파랑 */
  --color-accent-red-bg: 239 246 255;
  /* ... 나머지 토큰 */
}
```

**관련 파일**: `globals.css`, `tailwind.config.ts`, `SettingsBar.tsx`, `settingsStorage.ts`

### 3.2 헤더 리디자인 (page.tsx)

**작업량: 소 (30분)**

현재:
```
┌─────────────────────────────────┐
│ 🟠 [로고] ETF 포트폴리오 매니저  [💾][📥] │  ← 컬러 배경
└─────────────────────────────────┘
```

토스 스타일:
```
┌─────────────────────────────────┐
│ ETF 포트폴리오           [⚙️][💾] │  ← 흰 배경, 작은 아이콘
└─────────────────────────────────┘
```

변경 사항:
- `bg-primary` → `bg-card-bg` (흰 배경)
- 로고 제거 또는 축소
- 타이틀 텍스트 색상 → `text-foreground`
- 아이콘 버튼 최소화

### 3.3 자산 총액 히어로 카드 (신규 섹션 in page.tsx)

## 4. 변경하지 않는 것

| 항목 | 이유 |
|------|------|
| **데스크톱 테이블 레이아웃** | 데스크톱은 테이블이 효율적. 모바일만 변경 |
| **비즈니스 로직** (`lib/`) | UI 변경만. 계산/저장/API 로직 불변 |
| **테스트 코드** | UI 변경이 테스트에 미치는 영향은 최소 (data-testid 유지) |
| **기존 3개 테마** | Claude/Dark/Classic 유지, Toss 테마 **추가** |
| **Excel 내보내기** | 기능 불변 |

## 5. 실현 가능성 평가

### 5.1 기술적 가능성: **높음** ✅

| 근거 | 설명 |
|------|------|
| CSS 변수 기반 테마 | 이미 `globals.css`에 테마 변수 구조 완성. 새 테마 추가만 하면 됨 |
| 모바일/데스크톱 분리 | `md:hidden` / `hidden md:block`으로 이미 분리. 모바일만 수정 가능 |
| 컴포넌트 구조 | 각 컴포넌트가 props로 분리되어 있어 독립적 수정 가능 |
| Tailwind CSS | 유틸리티 클래스 변경만으로 대부분 구현 가능 |

## 6. 권장 구현 순서

```
Phase 1: 기반 작업 (Toss 테마 + 헤더)
├── globals.css: Toss 테마 변수 추가
├── tailwind.config.ts: 필요 시 확장
├── settingsStorage.ts: "toss" 테마명 추가
├── SettingsBar.tsx: Toss 테마 버튼 추가
└── page.tsx: 헤더 조건부 스타일

Phase 2: 핵심 화면 (히어로 + 종목)
├── page.tsx: 자산 총액 히어로 카드 (모바일)
```

## 7. 결론

| 항목 | 판단 |
|------|------|
| **실현 가능한가?** | **예.** 기존 아키텍처(CSS 변수 테마, 모바일/데스크톱 분리)가 토스 스타일 적용에 이미 최적화되어 있음 |
| **작업량이 많은가?** | **중간.** 약 10개 파일 수정. 신규 파일 생성은 0~1개. 각 파일당 모바일 영역(`md:hidden` 블록)만 수정 |
| **무엇이 변경되는가?** | 모바일 UI 전반 (색상 테마, 헤더, 카드 스타일, 숫자 크기, 여백, 하단 내비). **데스크톱과 비즈니스 로직은 불변** |
| **핵심 가치** | 기존 3개 테마 유지 + "Toss" 테마 추가로 사용자 선택권 보장. 모바일 UX만 토스 스타일로 전환 |
